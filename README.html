JWT Bearer Token Generator

Client instructions for generating a signed RS256 JWT assertion and exchanging it for a Salesforce access token. Compatible with both Sandbox and Production orgs.

✅ Prerequisites

Choose one runtime — all three use the same jwt.properties config file:

RuntimeRequirementVerify
☕ JavaJDK installedjavac -version
🟢 Node.jsNode.js installed — no extra packages needednode --version
🐍 PythonPython 3 + cryptography packagepython --version
  • ✓ Script file in the same folder as jwt.properties and your private key .key file
  • ✓ Private key in PKCS#8 PEM format — keys exported from Salesforce Certificate & Key Management are already in this format
  • ✓ Your org's My Domain URL — find it in Salesforce Setup → My Domain

⚙️ Configuration — jwt.properties

All JWT parameters are stored in jwt.properties. Update this file with your org values before running.

PropertyDescriptionWhere to find it
consumer_key ECA Consumer Key — JWT iss (issuer) Salesforce Setup → External Client Apps
username Salesforce integration username — JWT sub (subject) Salesforce Setup → Users
audience Your org's My Domain URL — JWT aud Salesforce Setup → My Domain
private_key Path to your RS256 PEM private key file e.g. server.key
expiry Token lifetime in seconds — Salesforce max is 180 180 (recommended)

🌐 Audience (aud) — My Domain URL

Set audience to your org's My Domain URL — this is the recommended approach per Salesforce documentation. Do not use the generic login.salesforce.com or test.salesforce.com endpoints.

🧪 Sandbox
[MyDomain]--[SandboxName].sandbox.my.salesforce.com
https://mycompany--qa.sandbox.my.salesforce.com
🚀 Production
[MyDomain].my.salesforce.com
https://mycompany.my.salesforce.com
ℹ️

Your My Domain URL can be found in Salesforce Setup → My Domain. Copy the full URL including https://.

💻 PowerShell Commands

Navigate to your JWT folder first, then choose a runtime below:

PowerShellPS
cd C:\path\to\your\JWT\folder
1
Compile — run once, or after changes to GenerateJWT.java
PowerShellPS
javac GenerateJWT.java
2
Generate the JWT assertion
StandardPS
java GenerateJWT --config jwt.properties
With verbose output — shows decoded claimsPS
java GenerateJWT --config jwt.properties --verbose
3
Exchange JWT for a Salesforce access token
Token ExchangePS
$JWT = java GenerateJWT --config jwt.properties

# POST JWT assertion to your org's token endpoint
$response = Invoke-RestMethod `
    -Method POST `
    -Uri "https://[your-org].my.salesforce.com/services/oauth2/token" `
    -Body "grant_type=urn:ietf:params:oauth:grant-type:jwt-bearer&assertion=$JWT" `
    -ContentType "application/x-www-form-urlencoded"

$accessToken = $response.access_token
$instanceUrl = $response.instance_url

🌐 Alternative: Generate JWT via jwt.io

If you prefer a browser-based approach without running any code, you can use jwt.io to manually build and sign the JWT assertion.

1
Open jwt.io and select JWT Encoder

Go to https://jwt.io → click the JWT Encoder tab → set algorithm to RS256.

2
Fill in the Payload

Replace the placeholder values with your values from jwt.properties:

PayloadJSON
{
  "iss": "<consumer_key from jwt.properties>",
  "sub": "<username from jwt.properties>",
  "aud": "<My Domain URL from jwt.properties>",
  "exp": 1773862391  // replace with current Unix time + 180
}
3
Calculate the exp value

Open your browser's developer console (F12 → Console) and run:

Browser ConsoleJS
Math.floor(Date.now() / 1000) + 180

Paste the result as the exp value in the payload.

4
Paste the Private Key

Under Sign JWT, set Private Key Format to PEM and paste the full contents of your server.key file.

Confirm all three green indicators appear: Valid header, Valid payload, Valid private key.

5
Copy the Encoded JWT

The signed JWT assertion appears in the Encoded JWT panel on the right. Copy it — this is your assertion for the Salesforce token exchange.

⚠️

jwt.io is a public tool. Do not paste production private keys into it. Use only for testing and validation with non-sensitive keys.

🔑 JWT Claims

The script automatically populates the following claims in the signed JWT:

iss — Issuer
ECA Consumer Key from jwt.properties
sub — Subject
Salesforce integration username from jwt.properties
aud — Audience
Your org's My Domain URL from jwt.properties
exp — Expiry
Current Unix time + 180 s — equivalent to Math.floor(Date.now()/1000) + 180

📦 Sample Token Response

After a successful token exchange, Salesforce returns a JSON response:

ResponseJSON
{
    "access_token": "<your-access-token>",
    "scope":        "web api",
    "instance_url": "https://[your-org].my.salesforce.com",
    "id":           "https://[login-url]/id/[org-id]/[user-id]",
    "token_type":   "Bearer"
}
FieldDescription
access_tokenThe token to pass in every Salesforce API request
instance_urlBase URL for all API calls — always use this, never hardcode the domain
token_typeAlways Bearer — used as the Authorization header prefix
scopePermissions granted to this token by the connected app

🚀 Using the Access Token

Pass access_token as a Bearer token in the Authorization header of every Salesforce API request.

SOQL Query — PowerShellPS
# Use $accessToken and $instanceUrl from the exchange step above
Invoke-RestMethod `
    -Method GET `
    -Uri "$instanceUrl/services/data/v59.0/query?q=SELECT+Id,Name+FROM+Account+LIMIT+5" `
    -Headers @{ Authorization = "Bearer $accessToken" }
⚠️

The JWT assertion is valid for 3 minutes only. Re-run the script to generate a fresh token whenever needed.

ℹ️

Always use instance_url from the token response as your API base URL — never hardcode your org domain.

📋 Notes

  • All three scripts share the same interface — --config jwt.properties works identically across Java, Node.js, and Python
  • Java: No external libraries — uses built-in java.security package. Recompile only needed when GenerateJWT.java changes
  • Node.js: No external packages — uses built-in crypto module. No compile step required
  • Python: Requires cryptography package (pip install cryptography). No compile step required
  • Private key must be in PKCS#8 PEM format — keys exported from Salesforce Certificate & Key Management are already in this format
  • JWT is printed to stdout — safe to pipe or capture into a PowerShell variable
  • Signing algorithm: RS256 (SHA-256 with RSA) — required by Salesforce JWT Bearer flow
  • Compatible with both Sandbox and Production orgs — set the correct My Domain URL in jwt.properties
